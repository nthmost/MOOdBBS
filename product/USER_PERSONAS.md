# MOOdBBS User Personas

## Primary Persona: The Wanderer (You)

### Demographics
- **Location:** San Francisco
- **Lifestyle:** Urban explorer, gamer, technologist
- **Gaming background:** Extensive RimWorld player, familiar with BBS culture
- **Tech comfort:** High - comfortable with code, APIs, automation

### Goals
- **Self-awareness without judgment** - Understand mood patterns objectively
- **Gamified exploration** - Make city exploration feel like quests
- **Quick logging** - Capture mood events with minimal friction
- **Pattern recognition** - See how activities affect mood over time

### Pain Points
- **Forgetting contributing factors** - "I feel bad but forgot all the little things"
- **Manual mood tracking friction** - Traditional journaling feels like work
- **Lack of exploration motivation** - Needs structure/gamification to explore
- **Generic self-care apps** - Don't speak the language of a gamer

### Usage Patterns
- **Morning:** Check mood status, understand weekly trends
- **Throughout day:** Quick event logging (10 second interactions)
- **Post-activity:** Quest completion + mood logging
- **Evening:** Review day's mood trajectory

### Key Quotes
- "Sometimes you know you feel kinda bad, but you forget all the little cuts and scrapes that got you to this point"
- "I want to work with you (Claude) to feel out how best to generate this content since I will definitely peter out on generating interesting quests"
- "I play Rimworld a lot and I just like the system"

### Must-Have Features
1. RimWorld-style mood modifiers
2. Numpad-based quick logging
3. Quest optionality (flexible goals, not rigid requirements)
4. Always-on dashboard showing current state
5. No GPS/location tracking (honor system)

### Nice-to-Have Features
1. Home automation integration (mood affects environment)
2. Voice assistant queries ("What's my current mood?")
3. Expedition log with digital artifacts
4. Memory system (RimWorld-style)

## Secondary Persona: The Self-Quantifier

### Demographics
- **Location:** Any urban area with walking/biking infrastructure
- **Lifestyle:** Data-driven self-improvement enthusiast
- **Gaming background:** May or may not be a gamer, but appreciates gamification
- **Tech comfort:** Medium to high

### Goals
- **Objective mood data** - Numbers and patterns over feelings
- **Habit formation** - Use quests to build healthy routines
- **Historical tracking** - See long-term mood trends
- **Goal achievement** - Complete quests, earn XP, level up (metaphorically)

### Pain Points
- **Mood tracking apps feel clinical** - Want something fun and visual
- **Hard to stick with journaling** - Too much friction, too time-consuming
- **Unclear cause-and-effect** - Don't know what actually affects mood
- **Generic fitness trackers** - Only track physical, not emotional/mental

### Usage Patterns
- **Daily check-ins** - Morning and evening mood logging
- **Post-activity logging** - After walks, social events, meals
- **Weekly reviews** - Check MoodStats to see patterns
- **Quest planning** - Select quests that align with goals

### Must-Have Features
1. Historical mood data and trends
2. Quest completion tracking
3. Clear cause-and-effect (modifiers visible)
4. Quick, frictionless logging

### Nice-to-Have Features
1. Recurring quest system (daily/weekly goals)
2. Export data for analysis
3. Integration with other tracking tools
4. Customizable mood modifiers

## Tertiary Persona: The Retro Enthusiast

### Demographics
- **Location:** Anywhere
- **Lifestyle:** Nostalgic for 90s tech, collector of vintage software
- **Gaming background:** Played BBS door games (LORD, TradeWars 2002, MajorMUD)
- **Tech comfort:** Medium - can run terminal apps, likes tinkering

### Goals
- **Aesthetic experience** - The 90s BBS feel is the main draw
- **Nostalgic computing** - Dedicated single-purpose device appeals
- **Unique software** - Tired of modern, flat, generic UIs
- **Conversation piece** - Cool to show friends

### Pain Points
- **Everything looks the same now** - Modern UIs are boring
- **No personality in apps** - Generic Material Design everywhere
- **Missing tactile computing** - Touch screens lack character
- **Lost sense of "logging in"** - No ritual, no sense of place

### Usage Patterns
- **Casual interaction** - More sporadic than primary persona
- **Appreciation of details** - Boot sequence, ANSI art, box characters
- **Customization** - Tweaking colors, ASCII art, MOTD
- **Showing off** - Demonstrating to friends

### Must-Have Features
1. Authentic 90s BBS aesthetic
2. Boot sequence with dial-in simulation
3. ANSI colors and ASCII art
4. MOTD and last login display
5. Dedicated hardware setup (always-on display)

### Nice-to-Have Features
1. Customizable ASCII art and themes
2. Additional retro touches (sound effects?)
3. Multi-user support (for the BBS feel)

## Anti-Persona: Not For Everyone

### Who MOOdBBS is NOT for:
- **Clinical mental health tracking** - This is not therapy or medical device
- **Team/workplace wellness** - Designed for single user, not enterprise
- **Children** - Assumes adult autonomy and urban mobility
- **People seeking positive psychology** - This is observation, not affirmations
- **Privacy-concerned users** - API allows LAN access, no encryption planned

## User Journey Map

### Discovery
- Hears about MOOdBBS from friend or Reddit/HN
- Attracted by RimWorld connection or BBS aesthetic
- Intrigued by "real world as ARG" concept

### Setup
- Downloads/installs on dedicated device
- Runs through initial setup (picks traits)
- Explores UI, tests numpad navigation
- Sees sample quests and mood events

### First Week
- Logs a few events, marks some quests complete
- Checks mood score frequently (novelty phase)
- Still figuring out what to log
- Adjusting to quick interaction pattern

### First Month
- Established logging habits
- Recognizes mood patterns
- Completed several quests
- Starting to customize modifiers and quests

### Sustained Use
- Daily check-ins feel natural
- Mood logging is reflexive after events
- Quest system motivates exploration
- Reviewing historical data provides insights
- Tweaking and personalizing the system

### Potential Drop-off Points
1. **Setup friction** - If initial configuration is too complex
2. **Empty state problem** - If quests/modifiers feel generic
3. **Novelty wears off** - If system doesn't provide ongoing value
4. **Logging friction** - If interactions take too long
5. **Lack of insights** - If mood data doesn't reveal patterns

### Retention Strategies
1. **Expedition Log** - Creates artifacts worth reviewing
2. **Memory system** - Generates nostalgia and reflection
3. **Quest variety** - Keep suggestions fresh and personal
4. **Seasonal variations** - MOTD, quest themes change over time
5. **API integrations** - Extend value beyond standalone use
